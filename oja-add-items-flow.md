## Key Design Principles

### 1. Single Input Bar
Text field, pantry button, and scan button all live in one bar.
No mode switching, no separate screens.

### 2. Converging Paths
All three input methods funnel into the same
suggestion â†’ edit â†’ add pipeline.

### 3. Zero-Tap Loop
After adding an item, focus returns to the input bar automatically.
Adding 10 items should feel like one uninterrupted flow.

### 4. Budget-First
Running total (spent vs budget) is always visible at the top.
Users see the financial impact before confirming each item.

### 5. Smart Defaults
Pre-fill everything possible from the pantry match or scan result.
The user only edits what's wrong â€” not re-enters what's right.

### 6. Inline Everything
No modal dialogs or page navigations. The edit card expands
in place below the input bar, keeping the list context visible.
